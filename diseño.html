Diseño de Software: Fundamentos

>_ Una infografía hacker sobre principios, elementos y metodologías del diseño de software

¿Qué es el Diseño de Software?

>_ Proceso de visionar y definir soluciones de software para problemas complejos

>_ Crear especificación de artefacto de software para cumplir objetivos

>_ Difiere del análisis: análisis descompone problemas, diseño enfoca capacidades

>_ Múltiples soluciones válidas para un mismo problema

Elementos Fundamentales
  • Diseño de datos
  • Diseño arquitectónico
  • Diseño de interfaz
  • Diseño a nivel de componente
  • Abstracción y refinamiento
  • Modularidad
  • Arquitectura de software
  • Estructura de datos
Importancia del Diseño

>_ La calidad es la palabra clave

  • Asegura la calidad interna del software
  • Impacta positivamente en todas las cualidades del sistema
  • Es crucial para el éxito del proyecto
  • Permite materializar requerimientos del cliente

>_ Cualidades del Software de Calidad

Mantenibilidad

>_ Capacidad para modificar el código fácilmente. Fundamental.

Claridad

>_ Código fácil de entender (opuesto a la "viscosidad").

Flexibilidad

>_ Permite modificar sin grandes esfuerzos (opuesto a la "rigidez").

Reusabilidad

>_ Facilita uso en otros proyectos con pocas modificaciones.

Robustez

>_ Opera bajo tensión o tolere entradas imprevistas.

Usabilidad

>_ Interfaz fácil de usar para su audiencia objetivo.

>_ Principios de Diseño según Davis

Evitar la "visión de túnel"

>_ Considerar enfoques alternativos y evaluarlos según requisitos y recursos.

Trazabilidad

>_ Diseño rastreable hasta el modelo de análisis.

No reinventar la rueda

>_ Utilizar patrones de diseño existentes.

Minimizar la "distancia intelectual"

>_ Estructura del diseño debe imitar el dominio del problema real.

Adaptación al cambio

>_ Diseño estructurado para permitir modificaciones futuras.

>_ Enfoques y Herramientas

Buenas Prácticas

  • Diseño Centrado en el Usuario (DCU): Involucra activamente al usuario
  • Prototipación: Versiones parciales para evaluación
  • Lenguajes de Modelado (UML): Describir comportamiento y estructura
  • Patrones de Diseño: Soluciones probadas para problemas comunes

Antipatrones a Evitar

  • Gran bola de lodo: Sistema sin estructura definida
  • Objeto todopoderoso: Demasiada funcionalidad en una clase
  • Código espagueti: Flujo de control confuso y complejo
  • Diseño en comité: Demasiados cocineros en la cocina

>_ Proceso de Diseño de Software

Análisis de Requisitos

>_ Comprender y documentar lo que el software debe hacer

Diseño Arquitectónico

>_ Definir la estructura global y los componentes del sistema

Diseño Detallado

>_ Especificar algoritmos, estructuras de datos y interfaces

Implementación

>_ Convertir el diseño en código ejecutable

Pruebas y Validación

>_ Verificar que el software cumple con los requisitos

Mantenimiento

>_ Realizar modificaciones y mejoras después del despliegue
